# AGENTS.md

エージェント(Claude Code / Codex など)共通の開発規約の正本。

## プロジェクト概要

127xx — ローカル開発ツールのためのポート台帳。ホストで LISTEN 中のポートを一覧し、任意のポートに名前・メモを付けて管理する。

名前の由来は `127.0.0.1` + `xx`。ポート `12700`〜`12799` を管理領域として扱う。

- `12700` — x127 自身の Web UI
- `12701`〜`12799` — 将来のランチャー機能の自動割当プール

**Status**: v0.1 実装中

## 技術スタック

- **言語**: Go(サーバ / CLI、シングルバイナリ)
- **UI**: Svelte(バイナリに埋め込み)
- **対応 OS**: macOS / Linux

## 開発コマンド

```sh
go build ./...      # ビルド
go test ./...       # テスト実行
go run .            # ローカル実行
gofmt -w .          # フォーマット
go vet ./...        # 静的解析
```

> Go コードは実装途上のため、コマンドは README で確定しているものを中心に記載している。ツール構成が固まり次第、随時更新すること。

## 設定・データ配置

設定・データは `~/.config/x127/` に保存する。

- `registry.json` — ポート台帳(名前・メモ)
- `x127.pid` — サーバプロセスの PID
- `x127.log` — ログ

## ポート規約

`12700`〜`12799` を x127 の管理領域とする。この範囲外のポートを既定で占有しないこと。

## 開発ワークフロー

- **issue 単位開発**: 作業は GitHub issue を起点に進める。作業前に issue を起票し、それに紐付けてブランチ・コミットを行う。
- **コミット形式**: `#N [type]: 概要`(`N` は issue 番号、`type` は `feat` / `fix` / `docs` / `chore` など)。
  - 例: `#7 [docs]: CLAUDE.md を追加`

## コーディング規約

- **フォーマット**: `gofmt` に従う。標準的な Go のレイアウト・命名慣習を踏襲する。
- **静的解析**: `go vet` を通す。
- **空行**: `gofmt` は関数内の空行を強制しないため、処理の塊ごとに適度な空行を入れて可読性を優先すること。

## コミュニケーション規約

- **言語**: レビュー・コミットメッセージ・ユーザーとのやり取りは、すべて日本語で行う。
